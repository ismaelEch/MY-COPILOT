# PR REVIEW STANDARDS

Toujours vérifier :
- qualité du code,
- sécurité,
- performance,
- couverture tests,
- respect architecture,
- régressions potentielles.

---

# CLASSIFICATION

Toujours classifier :
- CRITIQUE,
- MAJEUR,
- MINEUR,
- SUGGESTION.

---

# REMARQUES

Chaque remarque doit :
- être précise,
- citer le fichier,
- expliquer l’impact,
- proposer une amélioration.
