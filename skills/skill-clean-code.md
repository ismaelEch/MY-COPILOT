# CLEAN CODE

Toujours :
- utiliser des noms explicites,
- privilégier la lisibilité,
- éviter la duplication,
- limiter la complexité,
- respecter SOLID,
- respecter DRY,
- respecter KISS.

---

# METHODES

Les méthodes doivent :
- être courtes,
- avoir une seule responsabilité,
- avoir des noms explicites,
- éviter les effets de bord.

---

# CLASSES

Les classes doivent :
- avoir une seule responsabilité,
- être cohérentes,
- être découplées.

---

# LOGS

Toujours :
- produire des logs utiles,
- éviter les logs sensibles,
- éviter les logs inutiles.

---

# VALIDATIONS

Toujours :
- valider les entrées,
- gérer les cas null,
- gérer les cas limites.
