# API STANDARDS

Toujours respecter :
- conventions REST,
- conventions HTTP,
- conventions DTO,
- conventions pagination.

---

# ENDPOINTS

Les endpoints doivent :
- avoir des noms cohérents,
- respecter les verbes HTTP,
- respecter les conventions existantes.

---

# ERREURS

Toujours :
- utiliser des codes HTTP cohérents,
- retourner des erreurs structurées.

---

# PAGINATION

Toujours gérer :
- page,
- size,
- sorting,
- filtering.
